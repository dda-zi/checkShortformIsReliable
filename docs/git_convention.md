# Git 커밋 규칙

## 1. 목적
본 문서는 프로젝트 수행 과정에서 Git 커밋 메시지와 작업 단위를 일관되게 관리하기 위한 기준을 정의한다.  
커밋 이력을 명확하게 유지하여 변경 사항을 쉽게 추적하고, 산출물 및 소스코드의 관리 효율을 높이는 것을 목적으로 한다.

## 2. 기본 원칙
- 하나의 커밋에는 하나의 목적만 담는다.
- 커밋 메시지는 변경 내용을 명확하게 드러내도록 작성한다.
- 문서 수정, 기능 추가, 버그 수정, 구조 변경은 가능한 한 분리하여 커밋한다.
- 의미 없는 대규모 묶음 커밋은 지양한다.
